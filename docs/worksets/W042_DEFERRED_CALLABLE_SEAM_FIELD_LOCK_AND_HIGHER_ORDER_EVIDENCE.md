# WORKSET - Deferred Callable Seam Field Lock And Higher-Order Evidence (W42)

## 1. Purpose
Own the final current-phase callable-seam lock for the OxFml <-> OxFunc boundary.

This packet now exists to package one explicit shared freeze candidate for the admitted helper/callable surface rather than leaving the minimum carrier and richer-detail split as diffuse note traffic.

## 2. Provenance
Opened after:
1. the OxFml/OxFunc callable/library-context exchange converged on a provisional semantic direction,
2. `W38` established local empirical helper/lambda evidence through Stage 1 and Stage 2,
3. both repos initially chose not to force closure on the remaining callable transport and higher-order helper questions without stronger evidence.

Relevant context:
1. `docs/worksets/W038_FUNCTIONAL_LAMBDA_AND_HELPER_FAMILY.md`
2. `docs/function-lane/OXFML_OXFUNC_LET_LAMBDA_PIN_DOWN_RESPONSE_V1.md`
3. `docs/upstream/NOTES_FOR_OXFML.md`
4. `../OxFml/docs/upstream/NOTES_FOR_OXFUNC.md`

Current convergence floor:
1. typed invocation over opaque callable identity is now exercised on both sides for the currently admitted higher-order helper floor,
2. workbook Defined Name callable preservation is now real first-pass seam pressure rather than a late extension,
3. the minimum current-phase semantic callable fields are:
   - opaque callable identity,
   - origin kind,
   - capture mode,
   - arity shape,
   - invocation-contract meaning,
4. first-pass `ISOMITTED` omission semantics are now pinned locally enough that `ISOMITTED` no longer needs to drive the minimum-carrier freeze by itself,
5. richer callable detail for replay, explain, or serialization does not need to sit in the shared minimum carrier so long as OxFml and OxFunc can cooperatively preserve it when advanced scenarios require that richer detail,
6. only future broader callable transport, additional higher-order omission artifacts, or concrete implementation mismatches should require widening beyond the current minimum carrier.

## 3. Scope
This workset owns the current-phase callable freeze candidate for:
1. the minimum shared callable carrier:
   - opaque callable identity/token
   - `origin_kind`
   - `arity_shape`
   - `capture_mode`
   - `invocation_contract_ref`
2. typed invocation over the opaque callable token as the shared invocation boundary,
3. the split between the shared minimum carrier and richer provenance/replay detail,
4. helper-family pressure from `MAP`, `REDUCE`, `SCAN`, `BYROW`, `BYCOL`, `MAKEARRAY`, and `ISOMITTED` only insofar as those functions could prove the minimum carrier insufficient,
5. the statement that richer replay/explain/serialization detail may require explicit OxFml/OxFunc cooperation in advanced scenarios without forcing that richer detail into the minimum carrier.

## 4. Out Of Scope
1. a broad future callable ABI beyond the current admitted helper/callable surface,
2. forcing parameter-name lists, exact capture-name lists, helper body detail, or source spans into the shared minimum carrier,
3. requiring OxFunc alone to preserve richer callable replay/explain/serialization detail without OxFml assistance,
4. fuller UDF, add-in, and interoperable callable transport beyond the Excel-supported Defined Name callable surface,
5. general `@` / `SINGLE` placement work.

## 5. Entry Triggers
This workset should reopen only when one or more of the following occurs:
1. a proving-host or runtime artifact shows that the minimum shared carrier is insufficient,
2. `W38` or its successors produce higher-order helper evidence that materially changes the callable seam,
3. advanced replay/explain/serialization scenarios require richer callable detail and that detail cannot be handled through OxFml/OxFunc cooperative preservation outside the shared minimum carrier,
4. UDF/add-in/interoperable callable transport becomes implementation-ready enough to need a broader boundary lock,
5. callable/provider-stage behavior proves the generic staged availability model insufficient.

## 6. Expected Deliverables

3. one explicit statement that richer callable detail for replay/explain/serialization may be preserved cooperatively with OxFml in advanced scenarios without widening the minimum carrier,
4. any follow-on cross-repo handoff required to promote this freeze candidate into shared seam text.
