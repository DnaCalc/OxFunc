# WORKSET - Deferred Callable Seam Field Lock And Higher-Order Evidence (W42)

## 1. Purpose
Own the callable-seam questions that are now narrow enough to name clearly but do not yet have strong enough evidence to lock honestly in the OxFml <-> OxFunc boundary.

This packet exists so that the callable/library-context note exchange can close cleanly without leaving weakly evidenced callable questions as free-floating open prose.

1. the OxFml/OxFunc callable/library-context exchange converged on a provisional semantic direction,

3. both repos explicitly chose not to force closure on the remaining callable transport and higher-order helper questions without stronger evidence.

## 3. Scope
This workset owns future evidence-driven narrowing for:
1. minimum callable carrier field lock beyond the current candidate set,
2. final callable carrier versus provenance/replay split,
3. any need for an explicit invocation-model field beyond the current semantic invocation-reference idea,
4. higher-order helper seam pressure from `MAP`, `REDUCE`, `SCAN`, `BYROW`, `BYCOL`, `MAKEARRAY`, and `ISOMITTED`,
5. fuller defined-name, UDF, and interoperable callable transport,
6. any callable/provider-stage refinement beyond the generic staged availability model.

## 4. Out Of Scope
1. forcing canonical field names before the field set itself is stable enough,
2. treating OxFunc-local higher-order helper evidence alone as sufficient upstream seam-lock evidence,
3. reopening the callable/library-context note exchange without a concrete trigger,
4. general `@` / `SINGLE` placement work.

## 5. Entry Triggers
This workset should become active only when one or more of the following occurs:
1. OxFunc needs a concrete callable field set for implementation-facing boundary work,
2. a proving-host or runtime artifact forces a narrower callable carrier decision,
3. `W38` or its successors produce higher-order helper evidence that materially changes the callable seam,
4. defined-name/UDF/interoperable callable transport becomes implementation-ready enough to need explicit boundary locking,
5. callable/provider-stage behavior proves the generic staged availability model insufficient.

## 6. Expected Deliverables
1. a future callable-seam field-lock note,
2. one or more empirical higher-order helper packets beyond the current `W38` floor,
3. explicit callable carrier/provenance examples,
4. any cross-repo handoff required if coordinator-visible seam consequences emerge.
